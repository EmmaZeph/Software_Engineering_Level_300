Key differences between Agile and Waterfall Process Module.

Question 3
a. The Rational Unified Process is an iterative software development process framework created by the Rational Software Corporation, a division of IBM since 2003.

b. RUP defers from other software development process because of Iterative and incremental development: RUP emphasizes iterations, or cycles, of work, rather than trying to complete all work upfront. This allows for more flexibility and responsiveness to change. Each iteration results in a working version of the software that can be demonstrated to, and used by, clients.

c. RUP organizes a product release into four phases: initiation, elaboration, construction, and transition. 
        I. During initiation, project planners set the initial scope and budget for the project. 
        II. During elaboration, A detailed evaluation and development plan is carried out and diminishes the risks.
        III. Construction is the phase in which the bulk of the application is programmed.
        IV.  Transition is the phase in which The final project is released to the public.


Benefits of using RUP approach in software development projects:
        I. It provides good documentation, it completes the process in itself.
        II. It provides risk-management support.
        III. It reuses the components, and hence total time duration is less.
        IV. Good online support is available in the form of tutorials and training.

Drwbacks of using the RUP approach in software development projects:
        I. Team of expert professional is required, as the process is complex.
        II. Complex and not properly organized process.
        III. More dependency on risk management.
        IV. Hard to integrate again and again.
